# Contributing

Thanks for your interest in improving this project.

## Contribution basics
- Bug reports and feature requests: please open a GitHub issue.
- Pull requests: welcome, especially for fixes, tests, documentation, and small improvements.

For substantial changes (new features, architectural refactors, security-sensitive behavior), please open an issue first to discuss scope and approach. This avoids duplicated work and helps keep changes aligned with the project’s intent.

## Licensing of contributions (inbound = outbound)
This project is licensed under the Apache License, Version 2.0. By submitting a contribution (including via a pull request), you agree that your contribution is licensed under the same license (Apache-2.0).

Please do not submit code you do not have the right to contribute. If your change includes third-party material, you must clearly identify it and ensure it is compatible with Apache-2.0.

## Security issues
If you believe you have found a security vulnerability, please do not file a public issue. Use GitHub’s private security reporting for this repository (Security tab → “Report a vulnerability”), or open a draft security advisory.

## Style and quality
- Keep changes focused and easy to review.
- Add or update tests when behavior changes.
- Update documentation when user-facing behavior changes.
